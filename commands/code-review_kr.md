# 코드 리뷰

커밋되지 않은 변경사항에 대한 포괄적인 보안 및 품질 리뷰:

1. 변경된 파일 가져오기: git diff --name-only HEAD

2. 변경된 각 파일에 대해 다음을 확인:

**보안 문제 (CRITICAL):**
- 하드코딩된 자격 증명, API 키, 토큰
- SQL 인젝션 취약점
- XSS 취약점
- 누락된 입력 검증
- 안전하지 않은 의존성
- 경로 탐색 위험

**코드 품질 (HIGH):**
- 50줄 초과 함수
- 800줄 초과 파일
- 4레벨 초과 중첩 깊이
- 누락된 오류 처리
- console.log 문
- TODO/FIXME 주석
- 공개 API의 JSDoc 누락

**모범 사례 (MEDIUM):**
- 변경 패턴 (불변 사용)
- 코드/주석의 이모지 사용
- 새 코드에 대한 테스트 누락
- 접근성 문제 (a11y)

3. 다음을 포함한 보고서 생성:
   - 심각도: CRITICAL, HIGH, MEDIUM, LOW
   - 파일 위치 및 줄 번호
   - 문제 설명
   - 수정 제안

4. CRITICAL 또는 HIGH 문제가 발견되면 커밋 차단

보안 취약점이 있는 코드를 절대 승인하지 마세요!
