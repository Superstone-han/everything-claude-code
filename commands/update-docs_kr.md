1. package.json scripts 섹션 읽기
   - scripts 참조 테이블 생성
   - 주석의 설명 포함

2. .env.example 읽기
   - 모든 환경 변수 추출
   - 목적 및 형식 문서화

3. 다음으로 docs/CONTRIB.md 생성:
   - 개발 워크플로우
   - 사용 가능한 scripts
   - 환경 설정
   - 테스트 절차

4. 다음으로 docs/RUNBOOK.md 생성:
   - 배포 절차
   - 모니터링 및 알림
   - 일반적인 문제 및 수정사항
   - 롤백 절차

5. 오래된 문서 식별:
   - 90일 이상 수정되지 않은 문서 찾기
   - 수동 검토를 위해 나열

6. diff 요약 표시
