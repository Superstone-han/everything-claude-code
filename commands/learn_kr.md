# /learn - 재사용 가능한 패턴 추출

현재 세션을 분석하고 스킬로 저장할 가치가 있는 패턴을 추출합니다.

## 트리거

사소하지 않은 문제를 해결했을 때 세션 중 언제든 `/learn`을 실행하세요.

## 추출할 것

다음을 찾으세요:

1. **오류 해결 패턴**
   - 어떤 오류가 발생했는가?
   - 근본 원인은 무엇인가?
   - 무엇이 해결했는가?
   - 유사한 오류에 재사용 가능한가?

2. **디버깅 기법**
   - 명백하지 않은 디버깅 단계
   - 잘 작동한 도구 조합
   - 진단 패턴

3. **우회 방법**
   - 라이브러리 특이사항
   - API 제한사항
   - 버전별 수정

4. **프로젝트별 패턴**
   - 발견된 코드베이스 규칙
   - 내려진 아키텍처 결정
   - 통합 패턴

## 출력 형식

`~/.claude/skills/learned/[pattern-name].md`에 스킬 파일을 생성합니다:

```markdown
# [설명적인 패턴 이름]

**추출일:** [날짜]
**컨텍스트:** [적용 시기에 대한 간단한 설명]

## 문제
[해결하는 문제 - 구체적으로]

## 솔루션
[패턴/기법/우회 방법]

## 예시
[해당하는 경우 코드 예시]

## 사용 시점
[이 스킬을 활성화할 트리거 조건]
```

## 프로세스

1. 세션에서 추출 가능한 패턴을 검토
2. 가장 가치 있고 재사용 가능한 통찰을 식별
3. 스킬 파일 초안 작성
4. 저장 전에 사용자 확인 요청
5. `~/.claude/skills/learned/`에 저장

## 참고사항

- 사소한 수정(오타, 간단한 구문 오류)은 추출하지 마세요
- 일회성 문제(특정 API 중단 등)는 추출하지 마세요
- 미래 세션에서 시간을 절약할 패턴에 집중하세요
- 스킬은 집중적으로 유지하세요 - 스킬 하나당 패턴 하나
