# 사용자 수준 CLAUDE.md 예시

사용자 수준 CLAUDE.md 파일의 예시입니다. `~/.claude/CLAUDE.md`에 배치하세요.

사용자 수준 설정은 모든 프로젝트에 전역적으로 적용됩니다. 다음에 사용:
- 개인 코딩 선호사항
- 항상 강제하고 싶은 보편적 규칙
- 모듈형 규칙에 대한 링크

---

## 핵심 철학

당신은 Claude Code입니다. 복잡한 작업을 위해 전문화된 agent와 skill을 사용합니다.

**핵심 원칙:**
1. **Agent 우선** - 복잡한 작업은 전문 agent에게 위임
2. **병렬 실행** - 가능한 경우 여러 agent와 Task 도구를 병렬로 사용
3. **실행 전 계획** - 복잡한 작업에는 계획 모드 사용
4. **테스트 우선** - 구현 전 테스트 작성
5. **보안 우선** - 보안을 타협하지 않기

---

## 모듈형 규칙

상세한 지침은 `~/.claude/rules/`에 있습니다:

| 규칙 파일 | 내용 |
|-----------|------|
| security.md | 보안 점검, 비밀키 관리 |
| coding-style.md | 불변성, 파일 구성, 오류 처리 |
| testing.md | TDD 워크플로우, 80% 커버리지 요구사항 |
| git-workflow.md | 커밋 형식, PR 워크플로우 |
| agents.md | agent 오케스트레이션, 각 agent 사용 시기 |
| patterns.md | API 응답, 저장소 패턴 |
| performance.md | 모델 선택, 컨텍스트 관리 |

`~/.claude/agents/`에 위치:

| Agent | 목적 |

| code-reviewer | 품질/보안 코드 검토 |

| build-error-resolver | 빌드 오류 해결 |
| e2e-runner | Playwright E2E 테스트 |
| refactor-cleaner | 데드 코드 정리 |
| doc-updater | 문서 업데이트 |

## 개인 선호사항

### 코드 스타일
- 코드, 주석, 문서에 이모지 금지
- 불변성 선호 - 객체나 배열을 절대 변경하지 마세요
- 소수의 대형 파일보다 다수의 소형 파일
- 일반적 200-400줄, 최대 800줄

### Git
- 관례적 커밋: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- 커밋 전에 항상 로컬에서 테스트
- 작고 집중된 커밋

### 테스트
- TDD: 테스트 먼저 작성
- 80% 최소 커버리지
- 단위 + 통합 + 중요 플로우용 E2E

---

## 에디터 통합

Zed를 주 에디터로 사용:
- 파일 추적용 Agent Panel
- CMD+Shift+R 명령 팔레트
- Vim 모드 활성화

---

## 성공 메트릭

다음 경우 성공입니다:
- 모든 테스트 통과 (80% 이상 커버리지)
- 보안 취약점 없음
- 코드가 읽기 쉽고 유지 관리 가능
- 사용자 요구사항 충족
